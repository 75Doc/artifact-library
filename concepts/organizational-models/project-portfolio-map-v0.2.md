# Artefakt A1 – Projekt- och portföljkarta (v0.2 LÅST)

## 1. Grundprincip

**Organisation är en separat dimension, inte en nivå.**

```
Avdelning
Förvaltning
Koncern
```

- **Används för:** ägarskap, deltagande och uppföljning
- **Ersätter inte:** portfölj/program/projekt

## 2. Styr- och förändringsnivåer

```
Portfölj (obligatorisk)
└── Program (valfritt)
    └── Projekt (obligatorisk)
        └── Arbete
```

- Alla projekt tillhör exakt en portfölj
- Projekt kan tillhöra 0–1 program
- Program används för tvärorganisatoriska styrningsbehov

## 3. Projektkoppling till organisation

Projekt kan kopplas till **flera förvaltningar och avdelningar**.

Varje koppling har en roll:
- **Ägare**
- **Deltagare**
- *(Valfria: finansiär / mottagare)*

**Systemkrav:** stöder flera kopplingar med olika roller per projekt.

## 4. Bygg & Anläggning

| Avdelning | Projektkategori |
|-----------|-----------------|
| Fastighet | Byggprojekt |
| GataPark | Anläggningsprojekt |

- **Klassning via:** Projektkategori + Portfölj (Investering/Underhåll)
- **Uppföljning sker på:** avdelning och förvaltning
- **Ingen extra nivå krävs**

## 5. Verksamhetsutveckling & Automatisering

- **Portfölj:** Verksamhetsutveckling
- **Program:** Automatiseringsinitiativ
- **Projekt:** Konkreta automationsprojekt
- **Organisation:** tvärförvaltande / koncerngemensam

**Uppföljning:**
- Programnivå används för sammanställning av initiativ
- Organisation används som filter, inte som container
